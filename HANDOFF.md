# 引き継ぎガイド（Fableが使えなくなったとき用）

このドキュメントは、開発を担当していたAI（Claude Fable 5）が使えなくなった場合に、
後任のAIへ開発を引き継ぐための手順書。

## 誰に頼むか（推奨順）

1. **Claude Opus（その時点の最新版）** — 能力面でFableに最も近い。オーケストレーター役をそのまま任せられる
2. **Claude Sonnet（その時点の最新版）** — コスト重視ならこちら。これまでも実装担当のサブエージェントとして本プロジェクトのコードの大半を書いており、プロジェクトとの相性は実証済み
3. 上記の併用（メイン=Opus、実装サブエージェント=Sonnet）が現行体制の再現で理想

いずれも **Claude Code** で使うこと。Claude Codeはこのリポジトリの `CLAUDE.md` を起動時に自動で読むため、プロジェクトの流儀・パイプライン・禁止事項は自動的に引き継がれる。
（Claude Code以外のツールに移る場合は、最初に `CLAUDE.md` を読ませること）

モデル名は時期で変わるため「その時点で使える最上位のClaudeモデル」と読み替えてよい。

## 最初に貼る依頼文（テンプレート）

以下をコピーして、新しいセッションの最初に貼る。

---

もちぽよバトラーズというブラウザゲームの開発を引き継いでください。
前任のAI（Claude Fable 5）が使えなくなったため、あなたが後任です。

まず以下を読んでください：
1. このリポジトリの CLAUDE.md（体制・アーキテクチャ・作業ルールのすべて）
2. HANDOFF.md（このファイル。引き継ぎの背景）
3. git log --oneline -20（直近の開発履歴）

把握してほしい前提：
- 私はプロデューサーで、仕様書とイラストは「チャッピー」（ChatGPT）から私経由で届きます
- 実装はサブエージェント（Sonnet）に委譲し、あなた自身は仕様の整理・裁定と品質検証を担当してください（コスト方針）。サブエージェントには「再委譲禁止」を必ず明記
- 検証はあなた自身がブラウザ実測で行い、合格後にコミット→プッシュ（＝Vercel本番反映）→本番配信確認まで一括で行ってください
- チャッピーの仕様書には番号ズレや新旧用語の混在がよくあります。齟齬を見つけたら、あなたの裁定案を明示して私に確認してください
- UI文言はかわいく・ひらがな中心。体型を揶揄する表現は絶対に使わないでください

読み終わったら、現在のプロジェクト状態の要約と、CLAUDE.mdの「既知の仮置き」から見た次の一手の提案をください。
